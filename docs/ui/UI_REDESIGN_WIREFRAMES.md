# CellCap UI 개편 — 와이어프레임

> 이 문서는 [`UI_REDESIGN_PROPOSAL.md`](./UI_REDESIGN_PROPOSAL.md)의 §5 화면별 제안을 ASCII 와이어프레임으로 시각화한 보조 문서입니다.
> 기능·데이터·계약 변경 없음. 396×560pt 고정 팝오버 안에서의 시각 재정렬만 보여줍니다.
> 와이어프레임이라 색상·radius·픽셀은 정확히 묘사할 수 없고, **요소 위치·위계·정렬**에 집중합니다.
>
> 범례:
> - `[ Label ]` — 버튼 / 캡슐 배지
> - `( )` — 토글/체크
> - `▓▓▓░░░` — 진행바 채움
> - `◉` — 슬라이더 thumb, `│` — tick mark
> - `▾` `▸` — DisclosureGroup chevron
> - `⚡` `⏸` `⚠` `🔋` `ℹ` — SF Symbol 자리(개념적, 실제는 systemName)
> - 모든 텍스트는 현재 `MenuBarPresentation`이 만들어내는 한국어를 그대로 사용

## 1. 메뉴 막대 라벨

### 1.1 Before
```
 ┃ … 🔍 🔋 80%  🕐 14:32 ┃        ← macOS 메뉴 막대
                 ▲
                 └─ 일반 상태: 커스텀 CGPath 배터리(18×12) + 텍스트 "80%"

 ┃ … ⏸ 80% ┃                       ← suspended일 때만 SF Symbol 교체
 ┃ … ⚠ 80% ┃                       ← errorReadOnly일 때만
```
- 일반 상태(charging / holdingAtLimit / waitingForRecharge / temporaryOverride)에서 같은 아이콘 → 라벨만으로 모드 구분 불가.
- 텍스트 폭이 79→80→81로 흔들림(고정 폭 미적용).

### 1.2 After
```
 ┃ … 🔋⚡ 80% ┃           ← holdingAtLimit (battery.100.bolt)
 ┃ … ⚡🔋  80% ┃           ← charging / temporaryOverride (bolt.batteryblock)
 ┃ … 🔋·   80% ┃           ← waitingForRecharge (battery.75)
 ┃ … ⏸    80% ┃           ← suspended
 ┃ … ⚠    80% ┃           ← errorReadOnly
 ┃ … 🔋👁  80% ┃           ← read-only 모드 표식(eye 보조 글리프 inline)
```
- 5종 SF Symbol을 그대로 사용(`menuBarSymbolName` 이미 존재 — 커스텀 CGPath 통합 분기를 제거).
- `.monospacedDigit()`으로 % 흔들림 제거.
- accessibilityLabel = "배터리 80%, 상한 유지 중".
- 색은 시스템 메뉴 막대 톤 유지(.primary), 컬러 변경 금지.

## 13. 상태 매트릭스 한눈에 보기

> ChargeState × ControllerMode 조합에서 After 와이어프레임의 핵심 차이.

| ChargeState | fullControl (After) | readOnly / monitoringOnly (After) |
|---|---|---|
| `holdingAtLimit` | 정상: 모드 배지 숨김, 캡슐 0개, 고급 접힘 | 노랑 모드 배지, 정책 카드 잠금, 고급 자동 펼침 |
| `charging` | 진행바가 75 tick 통과 직후 색조 강해짐 | 동일 + 잠금 표시 |
| `waitingForRecharge` | 진행바 75 tick 아래에서 멈춤 | 동일 + ⚠ helper 도트 |
| `temporaryOverride` | 요약 카드에 "⚡ 1h 12m · [종료]" 인라인 | (이 상태로 진입 불가 — 권한 부족) |
| `suspended` | 회색 톤 카드 | 동일 |
| `errorReadOnly` | 빨강 stroke + 회복 액션 2개 인라인 | 동일 + Helper 권한 카드 |

## 14. 와이어프레임 한계

- ASCII는 색·블러·머티리얼·애니메이션(disclosure 펼침, 진행바 채움)을 표현할 수 없습니다. 실제 적용 시 라이트/다크 모드, Reduce Transparency에서의 검증이 필요합니다.
- `RangeSlider` 대안(§7.2 A)은 SwiftUI 기본 컴포넌트가 아니므로 시각만 합치고 입력은 두 `Slider`로 유지하는 안(§7.2′ B)이 안전합니다.
- 한국어 segmented `[30분│1시간│2시간│4시간]`의 폭은 396pt에서 실측 필요. 잘릴 경우 2×2 chip 그리드로 폴백.
- 카드/배지/도트의 색 톤은 [UI_REDESIGN_PROPOSAL.md §6 토큰](./UI_REDESIGN_PROPOSAL.md)을 기준으로 합니다 — 와이어프레임의 회색 음영은 톤 강도가 아니라 위계만 표현합니다.
- `summarySentence`, `temporaryOverrideSummaryText` 등은 `MenuBarPresentation`이 만드는 한국어를 그대로 사용합니다(개편 시 다듬을 수는 있으나 본 와이어프레임은 현재 문장을 보존).
